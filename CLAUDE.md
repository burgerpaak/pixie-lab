# Face Avatar Maker — 프로젝트 지침

## 핵심 원칙: SPEC 기반 개발

**모든 코드 변경은 반드시 SPEC.md를 기반으로 한다.**

### 워크플로우

1. **요구사항 변경 시**: SPEC.md를 먼저 수정 → 변경 이력 테이블에 기록 → 코드 수정
2. **버그 수정**: 스펙 변경 불필요 시 코드만 수정 (SPEC.md 변경 이력에 간단히 기록)
3. **새 기능 추가**: 반드시 SPEC.md에 기능 명세 추가 후 구현

### SPEC.md 수정이 필요한 경우 (주요 요구사항 변경)
- 새 카테고리 또는 옵션 추가/삭제
- 레이아웃 구조 변경
- 디자인 시스템 (컬러, 타이포, 인터랙션) 변경
- 새로운 기능 추가
- 반응형 브레이크포인트 변경
- 기술 스택 변경

### SPEC.md 수정이 불필요한 경우
- SVG 렌더링 버그 수정
- 브라우저 호환성 이슈 수정
- 코드 리팩토링 (동작 변경 없음)
- 성능 최적화

### 배포 확인
- 코드 변경 완료 후, 반드시 사용자에게 배포 여부를 확인한다
- 사용자가 승인하면 커밋 + push 진행 (GitHub Actions가 자동 배포)
- 배포 명령: `git add → git commit → git push`

### 변경 이력 기록 형식
SPEC.md 하단의 변경 이력 테이블에 다음 형식으로 기록:
```
| 날짜 | 버전 | 변경 내용 |
```

## 프로젝트 구조

```
face-avatar-maker/
├── CLAUDE.md          # 이 파일 (프로젝트 지침)
├── SPEC.md            # 요구사항 정의서 (Single Source of Truth)
└── index.html         # 단일 HTML 파일 (CSS + JS 포함)
```

## 기술 규칙

- **단일 파일**: 모든 CSS와 JS는 index.html 안에 포함
- **외부 의존성 없음**: Google Fonts 외 라이브러리 사용 금지
- **SVG 인라인**: 아바타 파츠는 JS 내 SVG 문자열로 관리
- **그레이스케일 전용**: 컬러 사용 금지 (피부 #E0E0E0, 눈동자 #333, 머리카락 #555, 입 #999)
